## Compatibility with Spool's MIT/ELv2 split

| Their license | Compatible with our MIT core? | Compatible with our ELv2 `/ee`? |
|---|---|---|
| MIT | ✅ yes | ✅ yes |
| ISC | ✅ yes | ✅ yes |
| Apache-2.0 | ✅ yes (with notice) | ✅ yes (with notice) |
| BSD-2 / BSD-3 | ✅ yes | ✅ yes |
| GPL / AGPL / SSPL / LGPL | ❌ would taint MIT core | ❌ would taint /ee |
| MPL-2.0 | ⚠️ file-level copyleft; case-by-case | ⚠️ case-by-case |
| (Multi-OR) | ✅ pick MIT/Apache/BSD branch | ✅ same |

The Apache-2.0 deps (`typescript`, `@hono/node-server` transitives in
some versions) require us to preserve NOTICE files when redistributing
binaries. We do that automatically through `npm install`'s standard
behavior; no special action required from contributors.

## How to re-run the audit

```bash
./scripts/license-audit.sh
```

The script:

1. Walks `node_modules` and tallies licenses for every installed package.
2. Fails (exit 1) if any **direct** dependency declares a license that's
   not in the allowlist (`MIT`, `ISC`, `Apache-2.0`, `BSD-2-Clause`,
   `BSD-3-Clause`, multi-OR strings containing one of these).
3. Warns (but doesn't fail) on transitive deps with non-allowlist
   licenses — copyleft contamination in a transitive only happens via a
   direct dep that pulled it in, which the first check would catch.
4. Treats the 4 known-benign undeclared paths above as audited.

When CI runs, this script is the gate. If it fails on your PR, the most
common fix is to swap the offending package for an MIT-licensed
alternative. If no alternative exists, open an issue first — we'll
decide whether to add an ELv2-internal use carveout (likely no) or
rework the feature to avoid the dep (likely yes).
